KPI-Aware Multimodal Anomaly Detection for 5G & Open RAN Telemetry

KAC is a reliable, deployment-oriented anomaly detector for high-dimensional telecom KPIs. This page showcases the paper, its key results, the pre-production deployment, and a fully reproducible code artifact with centralized logs and artifacts.

Paper

IEEE ICDM 2026 Applied Track.

IEEE ICDM 2026 · Applied Track

KPI-Aware Multimodal Anomaly Detection for 5G and Open RAN Telemetry

Chiman Salavati, Liang Wu, Kelly Wan, Mayank Darbari, Liangjie Hong
University of Connecticut · Nokia

KAC fuses full-KPI residual/uncertainty features from a frozen forecasting foundation model with compact operator-style text summaries, aligned by an uncertainty-weighted KPI-level contrastive objective. It improves threshold-based detection under severe class imbalance and strict alert budgets, with no online LLM inference at scoring time.

Read full abstract

Reliable anomaly detection is a deployment bottleneck for 5G/Open RAN observability and emerging self-optimizing 6G systems, where hundreds of heterogeneous KPIs, severe class imbalance, and strict alert budgets make purely numerical scores brittle. KAC turns compact operator-style summaries into a learnable feedback signal over full-KPI temporal evidence, giving the detector broad numerical coverage while retaining sparse KPI-specific reasoning. Across public telecom telemetry (TelecomTS), enterprise-scale Open RAN data (SpotLight), and real 5G RAN production packet traces (ProdTrace-SA), KAC consistently improves over residual-only, deep MTSAD, and zero-shot LLM baselines, and is integrated in pre-production shadow mode with Nokia's GKE-based KPI-construction pipeline.

Deployment: pre-production shadow mode

KAC is integrated in pre-production shadow mode with Nokia's GKE-based KPI-construction pipeline: it scores cached telemetry windows on CPU and logs scores for offline comparison against an internal detector. It does not promote incidents or raise operator-facing alerts. The repo ships a portable, CPU-only version of this path under deployment/.

  • Scorer — FastAPI service (/healthz, /readyz, /model-info, POST /v1/score); CPU-only, no online LLM calls.
  • Shadow harness — scores cached windows and appends a JSONL shadow log for offline comparison; never promotes incidents.
  • Operational-cost benchmark — regenerates the parameter breakdown (~66.5M params; KAC-specific ≈ 0.21%) and CPU latency (batch-1 median ≈ 15 ms, under the 100 ms telemetry cadence).
  • Container & orchestration — Dockerfile, docker-compose.yml (scorer + MinIO), and Kubernetes manifests with liveness/readiness probes and an HPA.
# Offline plumbing checks (no data / GPU / keys)
pip install -r deployment/requirements.txt
KAC_SMOKE=1 KAC_N_KPIS=6 uvicorn deployment.serve_app:app --port 8765
python -m deployment.shadow_runner --smoke        # -> logs/shadow_scores.jsonl
python -m deployment.benchmark_latency --smoke     # -> artifacts/deployment/

# Containerized scorer + cached-artifact store, or a cluster
docker compose -f deployment/docker-compose.yml up --build
kubectl apply -f deployment/k8s/

Reproduce

Every experiment runs through a single entry point, main.py. Outputs are written to artifacts/<run_id>/ and logs to logs/<run_id>.log. 

# Set up
pip install -r requirements.txt

# Fast offline sanity check (no data / GPU / API keys):
python main.py kac --dataset telecomts --smoke
pytest -q tests/

# Download public corpora + build the Chronos-2 residual cache
python scripts/download_telecomts.py
python scripts/download_spotlight.py
python scripts/compute_chronos_residuals.py --dataset telecomts

# Headline KAC (paper Table 2 / Table 4)
python main.py kac --dataset telecomts --scenario balanced --seeds 42 123 456 789 1337
python main.py kac --dataset spotlight

# Component ablation V1/V2/V3 (paper Table 6)
python main.py ablation --dataset telecomts --variants V1 V2 V3

# Baselines: deep MTSAD, frozen TSFM probe, zero-shot LLM
python main.py sota --dataset telecomts
python main.py foundation --dataset spotlight --foundation-model moment
python main.py llm --dataset telecomts --provider gpt

Override any setting with a dedicated flag (--epochs, --lr-head, --beta-kpi, ...) or the generic --set key=value. See the README for the full command list.
