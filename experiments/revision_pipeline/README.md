# KAC Strong-Accept Revision Pipeline

This folder contains every analysis script and runner used to produce
the revised IEEE ICDM 2026 KAC paper (`ICDM_2026_Applied_Track/paper/main_kac_paper.tex`). The
goal of the revision is to address the weaknesses raised in the
top-tier-conference reviewer report shipped with the project.

## What this folder produces

Running every script in numeric order (`00_paths.py` is a shared paths
module, `runner_*` scripts require a GPU) regenerates the intermediate
table and figure snippets used by the paper. Some historical scripts still
write into the old local draft directory names; update `00_paths.py` before
using them for a new submission bundle.

| Script | Purpose | Outputs |
|---|---|---|
| `01_build_ablation_tables.py` | 4-variant component ablation table | `tables/ablation.tex` |
| `02_build_headline_tables.py` | TelecomTS / SpotLight / ProdTrace-SA headline tables (5-seed mean$\pm$std for KAC, single-seed for baselines) | `tables/{telecomts,spotlight,production}_benchmark.tex` |
| `03_bootstrap_production.py` | Wilson + bootstrap 95% CIs for the 13-positive ProdTrace-SA test set | `tables/production_ci.tex` |
| `04_description_strategy.py` | SpotLight KPI-selection-strategy ablation | `tables/desc_strategy.tex` |
| `05_hp_sensitivity.py` | $(\alpha,\beta)$ heatmap from the existing 33-point grid | `figures_new/hp_sensitivity.pdf` |
| `06_threshold_sweep.py` | F1 vs. threshold for KAC ablation variants | `figures_new/threshold_sweep.pdf` |
| `07_ablation_figure.py` | Bar chart of V1$\to$V4 across all three datasets | `figures_new/ablation_per_dataset.pdf` |
| `08_significance.py` | Within-KAC Wilcoxon signed-rank tests + variance-reduction summary | `tables/variance_reduction.tex`, plus `wilcoxon_within_kac_holm.csv` |
| `09_deployment_params.py` | Parameter breakdown of the saved KAC checkpoint | `tables/deployment_params.tex` |
| `10_llm_table.py` | LLM zero-shot baseline table across all three datasets | `tables/llm_benchmark.tex` |

## Runners (require GPU and dataset access)

The runners below require an NVIDIA GPU (or Apple silicon MPS) and
local copies of the dataset splits + Chronos-2 residual cache.  They
are designed to be launched *from the dataset directory* so that the
relative paths to `data/features_cache/` resolve correctly, exactly as
the headline `KAC_Uncertainty_*_rawwidth.ipynb` notebooks expect.

| Runner | Workstream | Purpose |
|---|---|---|
| `runner_missing_baselines.py` | C | Train and evaluate text-only / residual-CNN / naive-fusion / MindTS-style / TS-CLIP-style baselines on all 3 datasets x 5 seeds |
| `runner_leakage_controls.py` | F | TelecomTS L1/L2/L3 text-leakage control conditions |
| `runner_deployment_bench.py` | D | Inference latency / throughput / parameter-count benchmark on a single device (GPU or CPU) |
| `runner_qualitative.py` | J | Forward-pass on saved KAC checkpoint to produce KPI-attention heatmaps and case studies |
| `runner_spotlight_pipeline_full.py` | K | Re-run the paper-aligned SpotLight pipeline on the same full-benchmark split as KAC |

Sample invocations:

```bash
cd evaluation_ver2/TelecomTS
python ../../ICDM_2026_Applied_Track/paper_build_scripts/runner_missing_baselines.py \
    --dataset TelecomTS --variants text-only residual-cnn naive-fusion \
                                   mindts-style tsclip-style \
    --seeds 42 123 456 789 1337

cd evaluation_ver2/TelecomTS
python ../../ICDM_2026_Applied_Track/paper_build_scripts/runner_leakage_controls.py \
    --condition L1 --seeds 42 123 456 789 1337

cd evaluation_ver2/TelecomTS
python ../../ICDM_2026_Applied_Track/paper_build_scripts/runner_deployment_bench.py
```

## Data provenance

* Per-seed KAC ablation (V1..V4 x 5 seeds x 3 datasets) is in
  `evaluation_ver2/<dataset>/results/kac_ablation.csv` and was
  generated by `evaluation_ver2/kac_ablation.py`.
* Per-seed SOTA baselines are in `evaluation_ver2/<dataset>/results/sota_<method>_per_seed.csv`
  but only seed 42 is shipped (single-seed). Re-running the runners
  above with multiple seeds is left to the camera-ready revision.
* Saved KAC checkpoints (`kac_uncertainty_best.pt`) are in
  `evaluation_ver2/<dataset>/results/`.

## Reproducibility checklist

* All KAC training runs use seeds 42, 123, 456, 789, 1337.
* All bootstrap CIs use 10,000 resamples with `numpy.random.default_rng(42)`.
* All Wilson CIs use the standard normal-approximation interval at $\alpha=0.05$.
* Hyperparameters per dataset are documented in `main.tex`, Section 5.1.5.
* Hardware: headline KAC runs use a single NVIDIA A100 (40GB);
  deployment-latency numbers in `tables/deployment_latency.tex` are
  CPU-only on an Apple M1.
